# Article Framework

Articles need a clear thesis, a deliberate sequence, and enough evidence to sustain attention. This framework is used before final voice adaptation.

## Universal Article Skeleton

```markdown
# Article Structure

## Working Title
[Plain title, not final headline.]

## Reader
[Who this is for and what they currently believe.]

## Thesis
[The core claim or answer.]

## Opening Contract
What the opening must establish:
Why the reader should care:

## Framework Choice
[Argument / Analysis / How-To / Narrative / Comparison / List]

## Body Sequence
1. [Beat: purpose, key point, evidence, transition]
2. [Beat: purpose, key point, evidence, transition]
3. [Beat: purpose, key point, evidence, transition]

## Counterargument or Complexity
[What a smart reader may object to.]

## Closing
What the piece should resolve:
What the reader should do or reconsider:
```

## Framework Choices

### Argument

Best for strong opinions and industry takes.

```markdown
Claim:
Why people believe the opposite:
Evidence 1:
Evidence 2:
Counterargument:
Resolution:
```

### Analysis

Best for explaining a pattern, trend, or system.

```markdown
Observed pattern:
Root causes:
Consequences:
What changes the interpretation:
Implications:
```

### How-To

Best for teaching a repeatable process.

```markdown
Problem:
Why common fixes fail:
Principles:
Steps:
Common mistakes:
Expected outcome:
```

### Narrative

Best when a story carries the insight.

```markdown
Scene:
Context:
Tension:
Turning point:
Meaning:
Transferable lesson:
```

### Comparison

Best for choosing between approaches.

```markdown
Decision context:
Option A:
Option B:
Tradeoffs:
Verdict:
When the verdict changes:
```

### List

Best for scannable insight collections.

```markdown
Premise:
Point 1:
Point 2:
Point 3:
Synthesis:
```

## Drafting Requirements

The Phase 3 draft should be exhaustive:

- every section's purpose
- each argument beat
- evidence or example placement
- transitions between sections
- counterargument handling
- approximate length allocation
- claims that must not be overstated

## Guardrails

- Do not write final prose.
- Do not apply voice.
- Do not invent facts.
- Do not bury the thesis.
- Do not include unsupported sections only because a framework has a slot.
