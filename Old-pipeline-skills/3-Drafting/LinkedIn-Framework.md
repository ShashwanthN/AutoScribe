# LinkedIn Structural Framework

Use this file to shape LinkedIn posts before final voice generation.

## Universal Skeleton

```markdown
# LinkedIn Structure

## Format
[Observation / Contrarian / Story / Practical / Data-led]

## Hook Direction
[What the first visible lines need to make the reader feel or question.]

## Reader Setup
[Who this is for and what they currently believe or experience.]

## Core Point
[The single idea the post must land.]

## Support
- [Example, story beat, data point, or concrete observation.]
- [Second support point if needed.]

## Progression
1. [Opening beat]
2. [Development beat]
3. [Turn or insight]
4. [Takeaway]

## CTA
[One action or question.]
```

## Format Selection

Use the user's material to pick a format:

- **Observation**: best when the strongest material is a pattern the writer has noticed.
- **Contrarian**: best when the piece challenges a common but flawed belief.
- **Story**: best when a concrete incident carries the point.
- **Practical**: best when the reader needs a usable method or checklist.
- **Data-led**: best when a number, benchmark, or result anchors the piece.

## Hook Direction Rules

In Phase 2, decide the hook direction. In Phase 3, provide hook options. Final wording waits for Phase 4.

Good hook directions:

- name a tension
- expose a hidden cost
- challenge a default assumption
- start at the moment of consequence
- make a specific promise

Weak hook directions:

- vague inspiration
- broad industry commentary
- generic "here are lessons"
- claims unsupported by the evidence bank

## Body Rules

- One idea per paragraph or bullet.
- Every beat must either add evidence, sharpen the claim, or move the reader toward the takeaway.
- Use concrete nouns from the ideation file.
- Keep platform constraints in mind, but do not write final prose yet.

## CTA Rules

Pick one:

- ask a genuine discussion question
- invite comments from a specific audience
- ask the reader to save/share only when the post is practical
- point to the next action if this is part of a larger content journey

Do not use multiple CTAs.
